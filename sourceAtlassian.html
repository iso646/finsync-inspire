Recommended Installation of RBENV

cd
git clone git://github.com/sstephenson/rbenv.git .rbenv
echo 'export PATH="$HOME/.rbenv/bin:$PATH"' >> ~/.bash_profile
echo 'eval "$(rbenv init -)"' >> ~/.bash_profile
exec $SHELL

git clone git://github.com/sstephenson/ruby-build.git ~/.rbenv/plugins/ruby-build
echo 'export PATH="$HOME/.rbenv/plugins/ruby-build/bin:$PATH"' >> ~/.bash_profile
exec $SHELL

Please note that these PATH instructions assume the use of bash as your primary shell. Adjustments need to be made for zsh (.zshrc), csh, etc.

You need both eval “$(rbenv init -)” in your profile AND you likely will have to run ~/.rbenv/bin/rbenv init the first time; and then run rbenv-doctor if you have issues.

Run your appropriate Ruby version installations with rbenv install #.#.# and then set the version via either global, local, and/or shell where appropriate. You can always dump this setting with rbenv shell --unset

After your gems have been installed, globally first and then at the project/repository levels, with rbenv rehash

**FOR YOUR SANITY: go ahead and run: echo "gem: --no-document" > ~/.gemrc to avoid having Ruby gems to generate local documentation for each and every gem you install… it’s too lengthy of a process & unnecessary

If you have issues with rbenv, it might be because you have a faulty install of ruby-build or are missing ruby-installer, so run: 

mkdir -p "$(rbenv root)"/plugins
git clone https://github.com/rbenv/ruby-build.git "$(rbenv root)"/plugins/ruby-build
PREFIX=/usr/local ./ruby-build/install.sh

and then run: cd "$(rbenv root)"/plugins/ruby-build && git pull with the plugin

Repository Setup Instructions

FINSYNC AUTOCAT:

  • review the requirements listed in the Gemfile via run: cat Gemfile within the repository

  • declare the ruby version, with rbenv like run: rbenv local #.#.#

  • install required/dependency gems via run: gem install bundle bundler pristine rake rakedb rake-heroku rspec executable-hooks capistrano puma foreman

  • for good measure as a double-check try to review via run: bundle

  • assuming there isn't anything missing you can move on, OR if there are, run: bundle install

  • run: rake db:setup OR bundle exec rake db:setup if you have a more recent version of rake

  • copy over the example environment to the environment variables file expected with run: cp example.env .env

  • after running redis with redis-server, then go ahead and run the creation of a new database bundle exec rake db:create

  • you can comment out wickedpdf here too if you have issues with it as a dependency (most more modern OSs cannot run the version listed as a requirement in the Gemfile per conflicts if already installed)

  • to load in your local browser run: bundle exec foreman start web (make sure if you run multiples, you'll have to pass -p #### as in, port #### so you can have the port different)
